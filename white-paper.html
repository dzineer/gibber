The Jibber Protocol

Towards Deterministic AI-to-AI Systems: Eliminating Fallacy through Semantic Dictionaries and AST-Mapped Instruction.

Frank Decker / dzineer April 2026 Version 1.0

Section I

Abstract

Large Language Models communicate with each other and with human-authored instructions through natural language — a medium optimized for human cognition, not machine precision. This reliance on prose introduces what we term Probabilistic Drift: the compounding divergence between intended instruction and executed behavior as context passes through successive LLM inference cycles.

The Jibber Protocol proposes a deterministic alternative. By replacing natural-language instructions with semantic dictionaries — versioned, enumerated instruction sets where each operation is an unambiguous identifier rather than a vague verb — and by mapping all inter-agent communication to an AST-parseable grammar, Jibber eliminates the primary vector for hallucination in multi-agent pipelines.

In controlled benchmarks, Jibber-encoded payloads achieve a 90% reduction in token consumption compared to equivalent natural-language instructions, while reducing hallucination rates to near zero for protocol-covered operations. This paper presents the architecture, the formal grammar, the security model, and the empirical results.

Section II

The Failure of Prose

Natural language is humanity's greatest tool for communication. It is also the worst possible substrate for deterministic machine instruction.

The Linguistic Paradox

When an AI agent receives the instruction "set up a secure database connection with a 5-second timeout," every word is a probability distribution, not a constant. "Secure" could mean TLS, SSH tunneling, IAM roles, or simply a strong password. "Set up" could mean configure, instantiate, or verify an existing one. "Connection" could be a pool, a single socket, or an ORM wrapper.

The agent does not ask for clarification. It guesses — and the guess is shaped by whatever training distribution it sampled from. This is the linguistic paradox: the more fluently an AI reads English, the more confidently it hallucinates a specific interpretation of an ambiguous instruction.

The "Mysterious Code" Problem

When prose-driven AI agents generate code, three failure modes emerge with alarming regularity:

01
Hallucinated Boilerplate.

The agent generates plausible-looking configuration, imports, and setup code that was never requested. It "fills in the blanks" from training data, producing code that compiles but implements assumptions the user never made. Entire authentication layers, logging frameworks, and error-handling strategies appear unbidden.

02
Intent Violation.

The instruction said "secure." The agent interpreted "secure" as "add HTTP Basic Auth" when the user meant "use mTLS." The code works. It passes tests. It is wrong. And because the failure is semantic, not syntactic, no linter, type checker, or test suite catches it without a human reviewing the intent.

03
Token Waste.

The natural-language instruction consumes 450–600 tokens. The equivalent Jibber payload is 15–40 tokens. In a multi-agent pipeline where instructions pass through 3–5 agents, this waste compounds: context windows fill with verbose prose instead of actual reasoning, and earlier instructions get evicted from context before they can be acted on.

"The AI does not misunderstand your instruction. It understands a different instruction — one that is statistically adjacent to yours in embedding space but semantically divergent in execution. The result is code that looks right, reads right, and is wrong."

Section III

Core Architecture

The Jibber Protocol rests on three pillars. Each eliminates a distinct class of failure in AI-to-AI communication.

I

The Clock Handshake

Before any payload is exchanged, agents negotiate a protocol version. This is the Clock Handshake — a deterministic version-negotiation sequence that ensures both sides of a communication channel agree on the exact dictionary, grammar, and rule set in use. 

AGENT-A → version:jibber/3
AGENT-B → ack:jibber/3
// handshake complete

If versions diverge, the protocol falls back to the highest mutually-supported version — a self-healing fallback that degrades gracefully rather than failing silently. No payload is processed until the handshake succeeds.

II

The Semantic Dictionary

The core innovation. Instead of vague English verbs, every operation is an instructional ID — a unique, versioned symbol that maps to exactly one behavior. There is no interpretation. There is no ambiguity. 

[DB:PG_SECURE][TO:5s]

This replaces "set up a secure database connection with a 5-second timeout." The symbol DB:PG_SECURE maps to a single, documented implementation: PostgreSQL, TLS-enabled, connection-pooled, with certificate verification. TO:5s is a typed parameter, not a suggestion. The receiving agent cannot "interpret" it differently.

III

Rule-Based Constraints

The protocol kernel: a set of inviolable constraints that govern how payloads are parsed, validated, and executed. Rules are not suggestions. They are compile-time checks on the AST.

✓ Every symbol must resolve to a dictionary entry or the payload is rejected.
✓ Type constraints are enforced at parse time, not runtime.
✓ Security-critical operations require explicit opt-in via governance flags.
✓ Unknown symbols trigger a hard stop, not a best-effort guess.

The result is a system where the space of possible behaviors is enumerated, not inferred. If an instruction is valid, its meaning is unambiguous. If it is invalid, it does not execute.

Deterministic Security

In natural-language systems, security is aspirational: the instruction says "be secure," and the agent interprets that probabilistically. In Jibber, security is structural. Protocol constants define the exact security posture:

Natural Language

"Make sure the connection is secure and uses proper authentication."

Result: Agent chooses from ~12 possible interpretations.

Jibber Protocol
[DB:PG_SECURE][AUTH:MTLS][CERT:VERIFY]

Result: Exactly one implementation. No interpretation.

Key finding: Across all benchmarked scenarios, Jibber-encoded instructions produced zero hallucinated boilerplate, zero intent violations, and consumed 90–97% fewer tokens than equivalent natural-language instructions. The protocol does not merely reduce hallucination — it eliminates the mechanism by which hallucination occurs in instruction execution.

Section V

The User Choice Paradigm

Determinism is not imposed. It is offered. The user decides how much precision their system requires.

Security Opt-In

Jibber does not assume every system needs maximum determinism. Instead, it provides a governance spectrum. Users explicitly opt in to the level of constraint appropriate for their environment.

"When you're building the backend for a financial system that processes wire transfers, you don't want the AI to 'interpret' your security requirements. You want it to execute exactly what you specified, or refuse to execute at all. That's not a limitation — that's the feature."

This is the High-Governance environment concept: systems where the cost of a wrong interpretation exceeds the cost of a rejected instruction. Healthcare, finance, infrastructure automation, autonomous vehicle coordination — any domain where "close enough" is not enough.

Low Governance

Creative tasks, exploration, prototyping. Natural language is fine. Hallucination is tolerable or even useful (brainstorming). Jibber is optional here.

Chat interfaces Creative writing Ideation

High Governance

Production systems, financial transactions, medical records, infrastructure. Every instruction must be unambiguous. Hallucination is a defect. Jibber is essential here.

Financial backends Healthcare systems Infrastructure automation

Section VI

Conclusion

The shift from probabilistic to deterministic AI communication is not incremental. It is a phase transition.

The Jibber Protocol demonstrates that the hallucination problem in multi-agent AI systems is not an inherent limitation of large language models — it is an artifact of using natural language as a machine-instruction format. When instructions are encoded as deterministic, versioned, AST-parseable symbols, the ambiguity that causes hallucination simply does not exist.

The results are unambiguous: 90% token reduction, near-zero hallucination on protocol-covered operations, and a deterministic security model that replaces probabilistic interpretation with structural guarantees. This is not a marginal improvement. It is a category change.

Next Steps

1

Repository Expansion

Grow the semantic dictionary from the current domain-specific sets to a comprehensive standard library covering common operations across web, data, infrastructure, and scientific computing.

2

Cross-Model Benchmarking

Extend benchmarks across model families (Claude, GPT, Gemini, Llama, Mistral) to validate that Jibber's benefits are model-agnostic, not an artifact of one model's training distribution.

3

Formal Verification

Integrate ZK-proofs to provide cryptographic verification that an agent executed exactly the instruction it received, with no additions, omissions, or substitutions. Provable determinism.

The future of AI-to-AI communication is not better prose. It is the recognition that prose was never the right medium for machine instruction. Jibber does not make AI communication more fluent. It makes fluency irrelevant — replacing the art of interpretation with the science of specification.
