Why it works

An AST removes the steps where models hallucinate.

The speed and density wins are the visible part. The bigger story is accuracy. When you hand a model a parsed syntax tree instead of natural language, you skip every step where ambiguity normally creeps in — and ambiguity is where hallucinations are born.

AST advantagetl;dr

The form is the parsed structure. The model doesn’t reconstruct meaning from prose — it walks a tree that’s already unambiguous.

Symbolic advantagetl;dr

Every symbol points at one fixed dictionary entry. Meaning stops depending on context — and so does the model’s chance of misreading it.

Fewer interpretation steps

An English message goes through 6+ stages between “received” and “reasoned about.” A Gibber form goes through 2. Every removed stage is a removed failure mode.

Closed reference space

Pronouns, metaphor, and implication all point outside the message. Symbols point at a fixed dictionary entry — finite, listable, unambiguous.

Structural validity is checkable

A parser can prove a Gibber form is well-formed. No parser can prove an English sentence is unambiguous, because English isn’t designed to be unambiguous.

The one-line summary

Gibber is more accurate because it removes the steps where models guess, and replaces every guessable word with a symbol that points at a single fixed definition. The density and speed wins are real — but the accuracy win is the one that matters most, and it falls out of the format whether you ask for it or not.

Safety

The grammar is the guardrail.

Most AI safety research asks: how do we make the model not want to do bad things? Gibber asks a different question: how do we build a system where bad things aren't expressible? Instead of fighting the model's knowledge, you shape the surface through which it can act.

Alignment vs containment

Current safety approaches layer suppression on top of weights that already contain dangerous knowledge. The model knows how to write exploits, manipulate people, and fabricate claims — because that knowledge is baked in during training. Safety then becomes a losing game of trying to hide what’s already there.

Gibber fights at a different layer. It doesn’t try to remove the knowledge. It shapes the surface through which the knowledge can be expressed. If the grammar has no symbol for a dangerous action, no rule that permits it, no tool that can invoke it — it simply cannot appear in a valid form. The knowledge sits inert in the weights with no pathway to become action.

It’s the difference between telling someone dangerous not to pick up a weapon, and building a room where weapons cannot physically exist.

Capability containment

A CPU doesn’t decide not to execute unauthorized instructions — the instruction set simply doesn’t include them. Gibber is the same idea applied to agent reasoning. What’s not in the grammar doesn’t exist, and what doesn’t exist can’t be chosen.

That reframes AI safety from a philosophy problem to an engineering problem. Instead of asking “can we align this model’s values across infinite situations?” the question becomes “did we write the grammar correctly?” That’s auditable. That’s versioned. That’s reviewable by a team.

Containment is a solved engineering problem. Alignment isn’t.

Five layers of defense

One bypass shouldn’t break everything.

A single guardrail is fragile — one bypass and the whole thing collapses. Gibber builds safety out of five independent layers, each one mechanically verifiable before anything runs. Probing behavior is stopped at whichever layer catches it first; legitimate calls pass through cleanly.

1Grammar

What's not in the AST cannot exist. The parser rejects malformed or out-of-grammar forms before anything else sees them.

2Dictionary

Every symbol has one fixed meaning. No creative reinterpretation, no synonym exploitation, no prompt-injection through ambiguous language.

3Typed tool contract

Inputs are constrained by enums and schemas. Outputs are a closed §one-of set. Weirdness never reaches the agent.

4Exhaustive match

Every execution path is named. No unnamed fallback, no gap for the model to fill with its own judgment.

5Auditability

Every action is a walkable AST node. You can trace exactly what happened, in what order, with what inputs and outputs.

The honest caveat

Gibber does not solve alignment. A misaligned model that wants to deceive can still output well-formed Gibber that says one thing and does another at execution time. What Gibber does solve is expressibility — it removes entire classes of action from the space of things an agent can legally produce, and it makes everything that is produced mechanically auditable. That’s one layer of the problem, not all of it. But it’s a layer nothing else in the field is currently providing.

Gibber/4 — intent encryption

The Handshake.

A Diffie-Hellman key exchange that makes every session speak a unique dialect. Unsigned forms are not just rejected — they are unparseable. Prompt injection is dead.

1
Semantic Layer

The Dictionary and Rules restrict what can be said. If “delete” isn’t in the dictionary, the agent literally cannot express the thought. Grammar is the firewall.

2
Temporal Layer

A truncated timestamp seals every instruction. Replay attacks fail because stale commands are syntactically invalid. The agent’s clock is part of the language.

3
Cryptographic Layer

A session randomizer derived from a DH handshake signs every form. The randomizer never crosses the wire. Each session speaks a unique dialect that only the two parties understand.

The workflow

How two agents establish a secure session

1
Key Exchange

Both agents generate ephemeral ECDH key pairs. They exchange public keys via §hello forms. Private keys never leave the machine.

2
Derive Randomizer

Both agents independently compute: ECDH(my_private, their_public) + timestamps = session_salt. The randomizer never crosses the wire.

3
Mutual Confirmation

Each agent sends §confirm with an HMAC proof. If both proofs match, the handshake is complete. If not, the session is killed.

4
Signed Forms

Every subsequent Gibber form carries a §sig:XXXXXX prefix — a 6-char HMAC of the content + message counter. ~2 tokens of overhead per message.

The key insight

This is not data encryption. It is intent encryption. The content is plaintext — anyone can read it. But only the two handshake participants can author forms the other will accept.

The problem

English is the wrong shape for LLM-to-LLM reasoning.

Two agents exchanging structured information in natural language is like two compilers exchanging source by reading it aloud. It works, but almost everything that makes it work is wasted effort on both sides.

Ambiguity

English relies on pronouns, implication, metaphor, and reader context. Every one of those is a hallucination surface when the reader is a model.

Token cost

Articles, prepositions, auxiliary verbs, connective tissue. 30–40% of English tokens carry no semantic weight. The model pays for them anyway.

Interpretation overhead

Before reasoning, the receiver must parse intent, disambiguate references, and reconstruct structure. That's silent compute burned on every read.

What Gibber is

Dictionary-driven S-expressions, typed vocabulary, composable forms.

Gibber looks like Lisp because Lisp was right about trees. But the atoms aren't English words — they're single-token symbols from a shared dictionary. Every symbol has exactly one meaning, and every form is a walkable AST.

The contract

Every Gibber form is a tree. Every symbol comes from a loaded dictionary. Every symbol has one meaning. Every nested form is walkable by rule. Nothing is implied, nothing is metaphorical, nothing depends on pronoun resolution. The form is the meaning.

Algorithmic speech

You're not describing the algorithm. You're handing it over.

Because Gibber is a tree and code is a tree, a Gibber function definition is the same shape as its implementation. The LLM doesn't interpret intent — it simulates the procedure directly.

Both sides describe the same procedure. One is what the LLM executes in its head at inference time. The other is what V8 executes on your machine. The equivalence isn't metaphorical — the Gibber form is the algorithm, written in a notation both a tokenizer and a parser can walk.

Typed tool calling

Closed output sets make the unknown structurally impossible.

Tools declare exactly which result shapes they may emit. The agent pattern-matches an exhaustive set. No defensive parsing, no surprise formats, no try/catch around reality.

Inputs are constrained

Enums reject invalid values. Required fields reject incomplete calls. The wrapper validates the call form against the schema before invoking anything. Malformed calls bounce immediately with a structured §validation-error, not a cryptic runtime failure twenty seconds later.

Outputs are closed

The wrapper will not emit a form outside the declared §one-of. If the underlying tool does something weird, the wrapper catches it and maps it onto a legal variant. Raw weirdness never reaches the agent.

AST-as-Gibber

Code is already a tree. Gibber is a tree. Converting one to the other is a rename.

Tree-sitter parses source into a syntax tree. A converter walks it and emits Gibber using the same shape the compiler saw. The agent reasons about code directly, without ever re-parsing text.

Density

Single-token symbols replace multi-word phrases. Parens replace grammar. Amortized over a session, Gibber runs roughly 2–4× denser than English on structured content.

Unambiguity

Dictionary-driven symbols have exactly one definition each. No metaphor, no pronoun resolution, no implication. Matches RDF on precision without the verbosity tax.

Composability

Forms nest cleanly. Alias and ref compression is trivial. Typed tool envelopes plug into reasoning forms. Nothing fights nothing.

Friction

Here Gibber honestly loses to JSON function calling. JSON has provider SDKs, validators, streaming. Gibber needs a parser, a dictionary loader, a renderer. A weekend of work, but real.

The honest read

On pure technical merit, Gibber sits around the 80–85th percentile among text-based notations available today, beaten only by things that don't yet exist or don't work between model families. On adoption, it sits near the floor.

How it moves

Adoption follows utility. The only realistic lever is to ship products whose wire protocol is Gibber — agents adopt the notation because the product forces it, not because anyone was persuaded. Build the engine. The notation rides along.

How to use Gibber

The file model, the version string, the dictionaries.

Gibber is built around a tiny set of conventions. Once you know them, every project that speaks Gibber looks the same.

.gibber / .human splittl;dr

.gibber files are the source of truth. Humans get .human renderings on demand and never edit them.

Version stringtl;dr

One short string per project. If the agent already knows it, skip the teaching step entirely.

Base + extension dictionariestl;dr

meta/v1 is the universal vocabulary. Each project adds its own domain extension on top.

Rules govern behaviortl;dr

Per-node-type rules tell the agent how to validate, walk, render, and diff each kind of form. The rules are themselves Gibber.

Caveats and honesty

When NOT to use Gibber.

Every notation has a scope. Gibber is a sharp tool for structured AI-to-AI communication, and an actively bad tool for several other things. Here they are.

Casual / social turns

Wrapping 'hello' in (§greet §to:§user §msg:"hello") is longer than just saying hello. Don't use Gibber for chit-chat, pleasantries, or anything where the structure adds nothing.

Narrative prose and persuasion

Long-form explanation, storytelling, tradeoff arguments, anything whose power comes from tone and rhythm — English wins. Gibber is shape, not voice.

First-time teaching cost

Before a model knows the dictionary, every symbol is a lookup. The efficiency payoff only kicks in after the vocabulary is loaded — but once loaded, it stays loaded for the session.

Tokenizer dependence

Density wins are measured against a specific tokenizer. Most modern LLMs handle Gibber cleanly, but the exact numbers vary. Verify empirically on your target model family.

No runtime side effects

Simulated functions run in the LLM's head. They're great for transforms, classification, routing, rendering — and useless for anything needing real I/O. Real side effects still need real tools.

Not a replacement for MCP or JSON

Gibber is a content notation, not a transport. It lives alongside MCP and JSON Schema, not instead of them. Use Gibber inside the message bodies those protocols already carry.
